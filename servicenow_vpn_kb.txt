Title: VPN Troubleshooting Guide

Applies To:
- ServiceNow Incidents
- Corporate VPN Users
- Remote Access Issues

Overview:
This knowledge base article provides standard troubleshooting steps for VPN-related incidents logged in ServiceNow. It is used by L1 and L2 support teams to resolve VPN connectivity issues.

Common VPN Incident Symptoms:
- User unable to connect to VPN
- VPN authentication failure
- VPN disconnects after login
- VPN connection is slow or unstable
- Backend applications unreachable after VPN connection

Preliminary Checks:
1. Verify the user has an active ServiceNow incident.
2. Confirm the incident category is set to "Network".
3. Confirm subcategory is set to "VPN".
4. Check user location and network type (home / office / public).
5. Verify the user's VPN account status in Active Directory.

Standard Troubleshooting Steps:
1. Ask the user to confirm internet connectivity without VPN.
2. Validate VPN credentials including username and password.
3. Check certificate validity if certificate-based authentication is used.
4. Ask the user to restart the VPN client.
5. Restart network services on the user's system.
6. Verify VPN client version is up to date.
7. Check firewall and antivirus interference.
8. Attempt connection from an alternate network if possible.
9. Validate VPN server status from monitoring dashboards.

ServiceNow Resolution Notes Example:
"User unable to connect to VPN. Verified internet connectivity. Restarted VPN client and network services. VPN connection restored successfully."

Escalation Criteria:
- VPN server unreachable
- Multiple users affected
- Authentication service failure
- Certificate expiry across users

Escalate such incidents to Network Operations (L3).

Related ServiceNow Fields:
